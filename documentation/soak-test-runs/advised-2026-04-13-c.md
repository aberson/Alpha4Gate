## Iteration 2

- **Selected improvement:** DEFEND hysteresis when attacking (rank #1, type: dev)
- **Principle(s) addressed:** §9 (seek favorable fights, not random fights)
- **What changed:** Added `DEFEND_INTERRUPT_THRESHOLD = 8` to `decision_engine._compute_next_state()`. When in ATTACK state, small enemy raids (< 8 supply near base) no longer pull the entire army back to defend. Prevents ATTACK↔DEFEND yo-yo pattern.
- **Root cause:** Any `enemy_army_near_base=True` triggered DEFEND, even a single unit. The army would march out, get recalled by a 2-supply raider, march out again — never reaching the enemy base.
- **Validation result:** PASS (marginal)
- **Metrics:** win rate 0/2 → 0/2, state changes reduced (Game 2: 45→18), army never reached ATTACK state in validation games (issue for iteration 3)
