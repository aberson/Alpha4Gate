```

POST /api/kill-daemons → {"results":["No daemon processes found"]}
GET /api/training/triggers → {"reason":"training already in progress"}
```

Daemon `running=False`, but an in-flight eval session kept claiming the "training in progress" flag. Over ~20 minutes the daemon ran **11 more eval games at diff 3** (all wins, extending the streak to **11/11**) before I gave up waiting and stopped it.

Saved feedback memory: [`feedback_daemon_eval_queue_persists.md`](../../memory/) documenting this behavior — the eval batch outlives the daemon stop, need to either wait it out or restart the whole backend.
