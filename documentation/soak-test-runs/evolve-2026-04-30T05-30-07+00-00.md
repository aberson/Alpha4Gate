## Postmortem (added 2026-04-30)

**Root cause:** WSL Ubuntu-22.04 distro had no `git user.email` / `user.name` configured. Every `git commit` in `git_commit_evo_auto` returned exit 128 with `fatal: empty ident name (for <abero@Apex.localdomain>) not allowed`, blocking all promotions. See [logs/evolve-linux-8h-20260429-2229.log:1759-1771](../../logs/evolve-linux-8h-20260429-2229.log) for the original stderr.

**Why the dashboard showed evictions:** `_apply_retry_bookkeeping` evicts `fitness-pass` imps once `retry_count >= _RETRY_CAP=3`. Because `commit_fail` does NOT change per-imp status (just fails the promotion), each strong winner kept being re-evaluated, kept passing fitness, kept retry-counting up — until the cap. The 6/9 and 7/9 winners shown in the dashboard's evicted list were *successful* candidates that the harness couldn't ship.

**Fix:** Run inside the WSL distro:
```bash
git config --global user.email "<your email>"
git config --global user.name "<your name>"
```

**Verified:** [logs/verify-commit-fix.sh](../../logs/verify-commit-fix.sh) (gitignored) directly invokes `git_commit_evo_auto` with a fake `bots/v5/` snapshot and now returns `(True, <sha>)` instead of `(False, None)`. See also [`documentation/wiki/linux-dev-environment.md`](../wiki/linux-dev-environment.md#7-configure-git-identity-required-for-evolve-commits) step 7.
