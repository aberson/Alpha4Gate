## User-reported bug (observed mid-iteration-2-validation)

Screenshot showed Stalkers warping into the main base trapped behind Nexus/Gateway/Pylon/Assimilator with no exit path. This is likely a contributing factor to losses: army can't deploy even when it wants to.

Root cause at `bot.py:676-681`:
```python
pos = await self.find_placement(
    ability, pylons.closest_to(self.start_location).position,
)
```
**Warp-ins always targeted the closest pylon to the main base** — the most crowded area. Forward pylons (natural, third) have more open space AND put units closer to fights.

## Iteration 3 — Warp-in forward pylon selection

### Fix

Modified warp-in pylon selection to iterate pylons **furthest-to-nearest** from start location:
```python
sorted_pylons = sorted(pylons, key=lambda p: -p.position.distance_to(self.start_location))
for pylon in sorted_pylons:
    candidate = await self.find_placement(ability, pylon.position, placement_step=2)
    if candidate is not None:
        pos = candidate
        break
```
- Tries forward pylons first (more open space, closer to fights)
- Falls back to main base only if all forward pylons fail placement
- `placement_step=2` uses tighter grid for more reliable placement
