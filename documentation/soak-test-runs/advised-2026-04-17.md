**Root cause remains unknown.** Across three morph variants (no filter, strict filter, loose filter) = 0 Archons across 30 games total. Likely `build_progress` returns 0 for units (it's really a structure attribute) — the filter excludes all HTs, confirming the pattern but not explaining iter 2's 0-Archon result where NO filter was used.

**Hypotheses for future iteration:**
1. The burnysc2 `unit(AbilityId.MORPH_ARCHON, other_unit)` call may need a different shape, such as ability-without-target or `unit.action(...)`.
2. Morph may require both HTs within a small distance (SC2 lore says no, but worth testing).
3. `subtract_cost=True` or `subtract_supply=True` kwargs may be required.
4. The HT may be auto-casting something (Feedback? unlikely without energy) that reserves the order slot.

**Next iteration test:** call `uv run python -m bots.v0.runner --batch 2 --map Simple64 --difficulty 3 --no-claude --decision-mode hybrid`, grep the game log for any `MORPH_ARCHON` dispatch line or error. Until we see the command actually land, the root cause is invisible.
