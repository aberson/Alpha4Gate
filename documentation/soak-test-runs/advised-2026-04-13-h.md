## Strategy: training-only workaround for the code bug

The bug at [bot.py:866-872](src/alpha4gate/bot.py#L866-L872) only activates `unit.move()` when `state NOT in (ATTACK, LATE_GAME)`. So: if PPO chooses the ATTACK state more often / earlier, the bug becomes inert for the ATTACK-state window.

Existing rules that push ATTACK-state selection:
- `attack-state-mid` +0.08 (after 4 min)
- `attack-state-late` +0.1 (after 6 min)
- `aggressive-finish` +0.03 (60+ supply attacking)

**This iteration's change:** increase ATTACK-state pressure in early/mid game to route around the move() bug.

**Validation batch: 0W-3L (0%).** Same win rate as prior-run hybrid baseline (also 0/3). Reward improved slightly (avg -103.9 vs -111.9 prior).

**Validation result: INCONCLUSIVE.**

Reason: PPO **inference** uses the frozen policy weights from `v1.zip`. Reward rule changes only influence weights **during PPO training** — they don't affect inference on an already-saved checkpoint. Both the prior hybrid run and this one used the same v1 policy, so the 0/3 was never going to shift without a retrain.

To actually validate, we'd need: daemon soak → PPO training epoch picks up new reward weights → new checkpoint saved → hybrid validation vs new checkpoint. The default `daemon_config.json` runs 5 cycles × 10 games at diff 1 (~50 games), which is beyond the 1-hour budget.

**The slight reward improvement (-103.9 vs -111.9)** is a weak positive signal — games are ending with marginally less net penalty — but not statistically meaningful at n=3.

## Iteration 1 — Disposition

**Keeping the changes.** Rationale:
- All four edits are directionally consistent with prior validated changes (e.g., `attack-state-mid` was doubled from 0.04 → 0.08 in "advised iter 3"; this is another +50% step in the same direction).
- No behavioral regression possible — these are reward signals consumed only by PPO training, and PPO training isn't even running.
- The only way they hurt is if, on a future daemon soak, they over-bias PPO toward premature attacks. That's testable next run.

No revert. No commit needed for `data/reward_rules.json` (gitignored). Only the log file commits.
