## Decision needed before Step 3

The `B → A` plan from the smoke 1 run log called for: "fix F2, re-smoke, then decide F1 path." Smoke 2 confirms the F2 fix works as intended. Now the F1 question:

The autonomous loop **still cannot train** because every cycle hits F1. Step 3 (the 4-hour soak) would now produce an honest dashboard full of "crashed" cards instead of fake green ones — that's an improvement but still not the goal of Phase 4.5 (which is "prove the loop runs unattended for hours **and trains something**").

**Three F1 fix paths, in order of effort:**

1. **Re-train from scratch** — discard all v1-v5 checkpoints, let the daemon's first cycle train a fresh model with the current 17-dim observation space. Lowest engineering effort, highest "real" cost (the v1 checkpoint represents prior training progress that gets thrown out).

2. **Detect and recover** — in trainer.py, catch the specific `ValueError` from `model.set_env` for shape mismatch and re-init the model from scratch instead of crashing. Requires distinguishing "this is a shape mismatch, restart fresh" from "this is some other ValueError, propagate." Mid-effort.

3. **Migrate old checkpoints** — extend the v1 weights with zeros for the 2 new dimensions and re-save. Highest engineering effort, requires understanding which 2 dimensions are new and where they fit in the observation vector. Most preserving of prior training.

**My recommendation: Option 1 (re-train from scratch).** Reasoning:
- Phase 4.5 explicitly says "soak test deliverable is the observation document, NOT the bot's strategic improvement." Throwing away v1's "skill" is not a real loss in Phase 4.5 terms.
- The v1 checkpoint is from a 15-dim observation space that no longer matches the env. It's already invalid.
- Option 2 papers over a real problem (silent obs space drift between checkpoint and env).
- Option 3 requires forensic investigation of what the 2 new dims represent — that's a side quest, not Phase 4.5 work.

**Suggested next move:** archive `data/checkpoints/v1.zip` to `~/soak-artifacts/pre-retrain/`, delete the entire `data/checkpoints/` directory, run smoke 3 to verify the daemon trains a fresh model from scratch, THEN run the 4-hour Step 3 soak.
