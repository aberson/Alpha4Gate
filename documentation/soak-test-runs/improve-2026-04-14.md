## Hypothesis (from code inspection)

Two dispatch points in [src/alpha4gate/bot.py](src/alpha4gate/bot.py) have no dedup / cooldown guard:

1. **WarpIn** — `_produce_army` [bot.py:661-701](src/alpha4gate/bot.py#L661-L701): iterates warpgates every step, calls `wg.warp_in(unit_id, pos)` whenever the ability shows available + can_afford + supply. Failed placement or race with cooldown activation causes immediate re-dispatch.
2. **Build** — `_drain_backlog` [bot.py:633-655](src/alpha4gate/bot.py#L633-L655) + `_build_structure` [bot.py:501-531](src/alpha4gate/bot.py#L501-L531): backlog entry stays queued and re-fires every tick when build command is dispatched but not yet reflected in `self.structures` (worker still walking, blocked placement, etc.).

### Code change summary

`DispatchGuard` in `src/alpha4gate/commands/dispatch_guard.py`:
- `should_dispatch(action, target, now) -> bool`
- `mark_dispatched(action, target, now) -> None`
- Per-action cooldown map (default: `WarpIn=2.0s`, `build=5.0s`), unknown actions fall back to `default_cooldown=2.0s`.

Wired into `src/alpha4gate/bot.py`:
- `_produce_army` WarpIn site ([bot.py:697-708](src/alpha4gate/bot.py#L697-L708)): guard denial breaks the inner ability loop so the outer warpgate loop advances.
- `_drain_backlog` build site ([bot.py:648-661](src/alpha4gate/bot.py#L648-L661)): guard denial returns early so the backlog entry is not drained or reappended.
- Guard constructed once in `Alpha4GateBot.__init__` ([bot.py:146](src/alpha4gate/bot.py#L146)). Bot is rebuilt per-game in `runner.py`, so no per-game reset hook is needed.

Tests: 6 new unit tests in [tests/commands/test_dispatch_guard.py](tests/commands/test_dispatch_guard.py). One collateral fix in `tests/test_bot_fortify.py` to inject the guard into the `MagicMock(spec=Alpha4GateBot)` fixture.

**Next action for the human:** When SC2 is next open for a dev game or a soak run, the first diff-4 log produced will either confirm the reduction (expected) or surface a regression. To check manually:

```powershell
uv run python -c "
import json
from collections import Counter
from pathlib import Path
log = sorted(Path('logs').glob('game_*.jsonl'))[-1]
by_target = Counter()
first = {}
last = {}
with open(log) as f:
    for line in f:
        e = json.loads(line)
        t = e.get('game_time_seconds', 0)
        for a in (e.get('actions_taken') or []):
            k = (a.get('action',''), a.get('target',''))
            by_target[k] += 1
            first.setdefault(k, t); last[k] = t
for (a,t), n in by_target.most_common(10):
    span = max(last[(a,t)] - first[(a,t)], 1)
    print(f'{n:>5}  {a:<10} {t:<25} rate={n/span*60:.1f}/min')
"
```

Pass: WarpIn/build rates per (action, target) ≤ 6/min for WarpIn and ≤ 12/min for build (≈10% of baseline). Fail: any rate above 15/min for WarpIn or 24/min for build — investigate whether the guard is wired correctly or if cooldowns need tuning.
